# 转化分析漏斗历史单日筛选设计

## 目标

调整 Dashboard“转化分析”漏斗的快捷时间筛选：

- 将“近1天”改名为“昨天”，保持其现有完整自然日语义。
- 在“昨天”后增加“前天”，查询前天单个完整自然日。
- 保持“当天”“近3天”“近一周”“近一月”“自定义”的现有行为不变。

## 日期语义

快捷单日范围以浏览器本地时区计算，并通过现有日期参数传给后端：

- 昨天：昨天 `00:00`（含）至今天 `00:00`（不含）。
- 前天：前天 `00:00`（含）至昨天 `00:00`（不含）。

请求仍使用包含首尾日期的 `startDate` 和 `endDate`。统计周期展示继续使用半开区间文本，明确实际边界。

## 实现方案

在 `funnelRange.mjs` 增加通用的历史单日范围构造函数，以“距今天的天数”为参数。`FunnelAnalytics.vue` 使用独立的 `yesterday`、`dayBeforeYesterday` 筛选值映射到偏移 1 天和 2 天，避免在组件内重复日期运算。

现有多日快捷范围继续使用 `buildCompletedDayRange`，当天继续使用 `buildCurrentDayRange`，自定义范围继续使用 `buildSelectedDayRange`。

## 测试与验收

先在 `tests/funnel-range.test.mjs` 增加失败测试，覆盖：

- 固定当前时间时，历史单日函数生成昨天和前天的正确日期与半开区间展示。
- 漏斗组件包含“昨天”和“前天”，且不再包含“近1天”。
- 两个筛选值均使用历史单日范围，不影响其他快捷范围。

实现后运行 `npm run test:unit` 和 `npm run build`。
