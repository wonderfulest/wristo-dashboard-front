# 应用管理页一键管理分类设计

## 目标

在管理后台的应用管理页提供一个作用于所有应用的分类维护入口，用于修复全量分类关系，不受当前分页、搜索或筛选条件影响。

## 交互规则

- 在应用管理页筛选工具栏增加“一键管理分类”按钮。
- 点击按钮打开弹窗，弹窗只要求管理员选择一个分类，不再额外选择操作类型。
- 选择 `whole` 分类时，唯一可执行操作为“加入所有应用”。
- 选择除 `whole` 外的任意分类时，唯一可执行操作为“清空该分类”，即从所有应用中移除所选分类；应用的其他分类关系保持不变。
- 弹窗根据所选分类展示将要执行的操作、作用范围“所有应用”和不可逆影响提示。
- 提交时二次确认。操作期间禁用重复提交并显示加载状态。
- 成功后展示实际新增或移除的分类关系数量，并刷新当前应用列表；失败时保留弹窗和选择，展示后端错误信息。

## 后端设计

- 在管理员产品接口中增加一个批量分类维护接口，接收所选分类 ID。
- 后端根据分类的 `slug` 决定操作，不接受前端传入任意操作类型：
  - `slug = whole`：为所有尚未关联 `whole` 的应用补充关系。
  - `slug != whole`：移除所有应用与该分类之间现存的关系。
- 接口仅允许管理员调用，并复用现有分类存在性、软删除和分类关系约定。
- 批量写入在事务中完成，避免部分应用已修改、部分应用未修改。
- 重复执行应保持幂等：已有 `whole` 关系不会重复创建，已清空的普通分类再次执行时影响数量为 0。
- 返回实际受影响的分类关系数量，供前端反馈。

## 前端设计

- 继续沿用 `Products.vue` 现有 Element Plus 弹窗、分类列表加载和消息反馈模式。
- 在 `src/api/products.ts` 增加对应的批量分类请求函数，组件不直接拼装 HTTP 请求。
- 分类列表为空或未选择分类时禁止提交。
- 操作文案由所选分类的 `slug` 派生，保证 `whole` 与普通分类不会出现错误按钮。

## 验证

- 后端测试覆盖：`whole` 批量补充、普通分类批量移除、重复执行幂等、分类不存在及非管理员拒绝访问。
- Dashboard 执行 `npm run build`，验证类型检查与生产构建。
- API 执行相关 Maven 定向测试；如全量测试存在无关基线失败，单独说明。
- 手工检查弹窗在 `whole` 和普通分类之间切换时的按钮与警告文案。

## 非目标

- 不按当前页或当前筛选结果限定作用范围。
- 不支持向所有应用批量加入普通分类。
- 不支持从所有应用批量移除 `whole`。
- 不改变现有单个应用的分类编辑能力。
